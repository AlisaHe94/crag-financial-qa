# GenAI Tool Disclosure

This project was developed with assistance from a generative AI tool.
This document discloses the tool, settings, scope of use, and points
to the prompts used.

## Tool

- **Name:** Claude (Anthropic)
- **Model:** Claude Opus 4.7
- **Interface:** Cowork desktop app, agent mode (multi-turn dialogue
  with file read/write and bash execution)
- **Settings:** Cowork-managed defaults; no custom system prompt
  beyond what the tool provides internally

No other AI tools (Cursor, Copilot, ChatGPT, Gemini, etc.) were used
during code authoring. Note: Gemini and Llama-3.1-8B are runtime
generators *inside* the system being demonstrated — that is documented
separately in the README and `.env.example`.

## Scope of use

The team set the project direction, made all design decisions, ran
experiments, and validated results. AI was used as a coding assistant
for:

- Drafting and debugging Python implementation files
- Drafting parts of project documentation (README, proposal sections,
  this disclosure, and selected report-section drafts), reviewed and
  edited by the team
- Drafting the initial 22-question evaluation set
  (`data/eval_questions.json`), reviewed by the team
- Refining specific slides in the human-authored slide deck
- Implementing the consistency-focused architectural iteration
  documented in `prompts/06_iteration_v2_v3.md` (modality-aware
  decomposition, numerical fidelity check, web-suppression toggle,
  α-ensemble inference, parallel judge runner, citation-correctness
  metric, qualitative comparison script), reviewed and validated by
  the team

## Prompts

Multi-turn dialogue cannot be reduced to single literal prompts.
Reconstructed summaries of the prompts used for each major code
component are in the [`prompts/`](./prompts/) directory. The committed
source code reflects the team-validated final state after iterative
review, debugging, and modification — it differs from any single AI
generation. This is standard practice in iterative software
development: prompts capture intent; code captures the team-validated
implementation.

| File | Code component |
|---|---|
| [`prompts/01_data_pipeline.md`](./prompts/01_data_pipeline.md) | `data_fetcher.py`, `document_processor.py` |
| [`prompts/02_baseline_rag.md`](./prompts/02_baseline_rag.md) | `rag_baseline.py` |
| [`prompts/03_crag_pipeline.md`](./prompts/03_crag_pipeline.md) | `crag_pipeline.py` |
| [`prompts/04_evaluation.md`](./prompts/04_evaluation.md) | `evaluate.py`, `judge_results.py` |

| [`prompts/06_iteration_v2_v3.md`](./prompts/06_iteration_v2_v3.md) | Consistency-interventions iteration in `v2/` and `v3/` |

Reproduction note: equivalent code can be regenerated by giving the
prompts above to a frontier-class LLM with file-editing capability.
Specific outputs (variable names, comment styles, edge cases) will
differ across runs even with identical prompts.
